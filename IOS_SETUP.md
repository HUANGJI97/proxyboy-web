# 📱 iOS 证书安装与信任详细指南

## 概述

本文档提供 iOS (iPhone/iPad) 设备安装和信任代理 CA 证书的详细步骤。

---

## 📋 完整安装流程

### 步骤 1：下载 CA 证书

**方法 1：通过 AirDrop（推荐）**

1. 在 Mac 上访问：http://115.159.196.184:8888
2. 点击"下载 CA 证书"按钮
3. 下载 `ProxySniffer_CA.crt` 文件
4. 右键文件 → **共享** → **AirDrop** → 选择你的 iPhone/iPad
5. iOS 设备会收到 AirDrop 通知，点击 **"接受"**

**方法 2：通过邮件**

1. 在电脑上访问：http://115.159.196.184:8888
2. 下载 `ProxySniffer_CA.crt` 文件
3. 作为邮件附件发送到自己的邮箱
4. 在 iOS 设备上打开邮件
5. 点击附件下载

**方法 3：直接在 iOS 设备上下载**

1. 在 iPhone/iPad 的 Safari 中访问：http://115.159.196.184:8888
2. 点击"下载 CA 证书"按钮
3. 证书文件会自动下载

---

### 步骤 2：安装描述文件

下载完成后，iOS 会自动打开"安装描述文件"页面。

**如果未自动打开：**

1. 打开 **设置** 应用
2. 点击 **通用**
3. 找到 **"VPN与设备管理"** 或 **"描述文件"**
   - iOS 16+：设置 → 通用 → VPN与设备管理
   - iOS 15 及以下：设置 → 通用 → 描述文件
4. 点击 **ProxySniffer CA**
5. 点击右上角的 **"安装"** 按钮
6. 输入 **设备密码**（锁屏密码）
7. 再次点击 **"安装"** 确认
8. 显示"已验证"说明安装成功

**截图示意：**
```
设置
└── 通用
    └── VPN与设备管理
        └── ProxySniffer CA
            └── [安装] → 输入密码 → [安装] → 完成
```

---

### 步骤 3：⚠️ 启用完全信任（非常重要！）

这是**最容易遗漏**的步骤！安装证书后，还必须启用"完全信任"，否则 HTTPS 解密无法工作。

**操作步骤：**

1. 打开 **设置** 应用
2. 点击 **通用**
3. 滑到最下方，点击 **"关于本机"**
4. 找到 **"证书信任设置"** 并点击
   - 如果找不到，在"通用"页面搜索"证书信任"
5. 在"对根证书启用完全信任"部分，找到 **ProxySniffer CA**
6. 打开右侧的 **开关**
7. 弹出警告框，点击 **"继续"**

**截图示意：**
```
设置
└── 通用
    └── 关于本机
        └── 证书信任设置
            └── ProxySniffer CA: [开关] → 打开 → [继续]
```

**验证是否启用成功：**

- 开关显示为 **绿色**
- 下方显示"已启用对根证书的完全信任"

---

### 步骤 4：配置代理

现在需要配置 iOS 设备的代理设置。

**操作步骤：**

1. 打开 **设置** 应用
2. 点击 **Wi-Fi**
3. 找到当前连接的 Wi-Fi 网络
4. 点击右侧的 **ⓘ (信息图标)**
5. 滑到最下方，找到 **"配置代理"**
6. 点击进入，选择 **"手动"**
7. 填写代理信息：
   - **服务器**：`115.159.196.184`
   - **端口**：`8080`
8. 点击右上角 **"存储"**

**截图示意：**
```
设置
└── Wi-Fi
    └── 当前 Wi-Fi ⓘ
        └── 配置代理
            └── 手动
                服务器: 115.159.196.184
                端口: 8080
                [存储]
```

---

### 步骤 5：测试

配置完成后，测试是否正常工作。

**测试方法：**

1. 打开 **Safari** 浏览器
2. 访问：`https://example.com`
3. 如果能正常打开页面，说明配置成功！
4. 如果显示"无法连接"或"证书错误"，请检查以下步骤。

**在 Web 管理端验证：**

1. 访问：http://115.159.196.184:3000
2. 应该能看到刚才访问 `https://example.com` 的抓包记录
3. 点击记录，查看解密的 HTTPS 数据

## ❌ 常见问题排查

### 问题 1：找不到"描述文件"或"VPN与设备管理"

**原因**：iOS 版本不同，位置可能不同。

**解决方法**：
- iOS 16+：设置 → 通用 → **VPN与设备管理**
- iOS 15 及以下：设置 → 通用 → **描述文件**
- 如果还是找不到，在"通用"页面顶部搜索框输入"描述文件"或"VPN"

---

### 问题 2：安装证书后，HTTPS 还是无法解密

**原因**：没有启用"完全信任"！

**解决方法**：
1. 设置 → 通用 → 关于本机 → 证书信任设置
2. 确保 ProxySniffer CA 的开关是 **绿色（打开）**
3. 如果已经是打开状态，尝试关闭再重新打开

---

### 问题 3：启用完全信任时，提示"无法验证此证书"

**原因**：这是正常的！因为是自签名证书，iOS 会提示无法验证。

**解决方法**：
- 直接点击 **"继续"** 即可
- 这是预期行为，不影响使用

---

### 问题 4：Safari 显示"无法连接"或"超时"

**可能原因**：
1. 代理服务未运行
2. 代理地址或端口填错
3. 网络防火墙阻止

**解决方法**：
1. 检查代理服务是否运行：
   - 访问 http://115.159.196.184:3000 查看状态
2. 检查代理配置：
   - 服务器：`115.159.196.184`
   - 端口：`8080`
3. 尝试关闭 Wi-Fi，再重新连接
4. 重启 Safari 应用

---

### 问题 5：安装后想卸载证书

**卸载步骤**：

1. 打开 **设置** → **通用**
2. 点击 **VPN与设备管理**
3. 找到 **ProxySniffer CA**
4. 点击进入，滑到最下方
5. 点击 **"删除描述文件"**
6. 输入设备密码确认

---

### 问题 6：Web 管理端显示"证书不受信任"

**原因**：iOS 还没有完全信任此证书。

**解决方法**：
1. 按照"步骤 3"重新检查"完全信任"设置
2. 确保开关是绿色
3. 重启 Safari 应用
4. 如果还是不行，尝试重启设备

## 🔧 高级配置

### 使用不同的 Wi-Fi 网络

如果切换到其他 Wi-Fi 网络，需要**重新配置代理**：

1. 设置 → Wi-Fi
2. 点击新连接的 Wi-Fi 的 ⓘ 图标
3. 配置代理 → 手动
4. 填写相同的代理信息

### 使用蜂窝网络（4G/5G）

代理设置**仅对 Wi-Fi 有效**，蜂窝网络无法使用 HTTP 代理。

**解决方法**：
- 使用支持代理的 VPN 应用
- 或仅在使用 Wi-Fi 时进行抓包测试
