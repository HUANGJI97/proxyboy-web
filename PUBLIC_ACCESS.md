## 下一步：配置防火墙/安全组

### 如果是腾讯云/阿里云/华为云等云服务器

你需要在云服务商的控制台配置**安全组规则**，开放以下端口：

| 端口 | 用途 | 协议 | 建议来源 |
|------|------|------|----------|
| 8080 | HTTP/HTTPS 代理服务 | TCP | 0.0.0.0/0（或指定 IP） |
| 3000 | Web 管理端 | TCP | 你的 IP（或 0.0.0.0/0） |
| 8888 | 证书下载服务 | TCP | 0.0.0.0/0 |

#### 腾讯云配置步骤：

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 进入 **云服务器 CVM** → **安全组**
3. 找到当前服务器使用的安全组，点击 **修改规则**
4. 添加入站规则：

```
协议：TCP
端口：8080
来源：0.0.0.0/0
策略：允许
备注：代理服务
```

重复添加 3000 和 8888 端口。

5. 点击 **完成**

#### 阿里云配置步骤：

1. 登录 [阿里云控制台](https://ecs.console.aliyun.com/)
2. 进入 **云服务器 ECS** → **网络与安全** → **安全组**
3. 找到当前服务器使用的安全组，点击 **配置规则**
4. 点击 **添加规则**：

```
协议类型：TCP
端口范围：8080/8080
授权对象：0.0.0.0/0
优先级：1
授权策略：允许
```

重复添加 3000 和 8888 端口。

5. 点击 **确定**

### 如果是物理服务器或 VPS（使用 iptables/ufw）

#### 使用 ufw（Ubuntu 防火墙）

```bash
# 开放端口
sudo ufw allow 8080/tcp
sudo ufw allow 3000/tcp
sudo ufw allow 8888/tcp

# 重新加载
sudo ufw reload

# 查看状态
sudo ufw status
```

#### 使用 iptables

```bash
# 开放端口
sudo iptables -A INPUT -p tcp --dport 8080 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 3000 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 8888 -j ACCEPT

# 保存规则（Ubuntu）
sudo iptables-save | sudo tee /etc/iptables/rules.v4

# 保存规则（CentOS）
sudo service iptables save
```

## 安全建议 ⚠️

### 1. 限制访问来源（推荐）

不要将端口开放给 `0.0.0.0/0`（所有人），而是仅允许特定 IP：

**示例：仅允许你的家庭/公司 IP 访问 Web 管理端**

```
来源：1.2.3.4/32  # 替换为你的公网 IP
端口：3000
```

**查询你的公网 IP**：访问 https://ip.sb 或 https://ifconfig.me

### 2. 使用密码认证

当前 Web 管理端没有密码保护，建议添加：

- 使用 nginx 反向代理 + 基础认证
- 或在代码中添加登录功能

### 3. 使用 HTTPS

公网传输建议加密：

- 为 Web 管理端配置 SSL 证书（Let's Encrypt）
- 使用 nginx 反向代理 + HTTPS

### 4. 监控异常访问

```bash
# 查看 Web 服务访问日志
tail -f /tmp/web-server.log

# 查看代理服务日志
tail -f /tmp/proxy.log

# 查看异常连接
netstat -tnp | grep -E ':(3000|8080|8888)'
```

## 故障排查

### 问题 1：无法访问公网 IP

**可能原因：**
1. 安全组未配置
2. 服务器防火墙未开放端口
3. 服务未监听 `0.0.0.0`

**解决办法：**
```bash
# 1. 检查服务监听地址
netstat -tlnp | grep -E ':(3000|8080|8888)'

# 2. 检查服务器防火墙
sudo ufw status
# 或
sudo iptables -L -n

# 3. 检查云服务器安全组（需要在控制台查看）
```

### 问题 2：连接超时

```bash
# 测试端口是否开放
telnet 115.159.196.184 3000

# 或使用 nc
nc -zv 115.159.196.184 3000
```

如果连接失败，说明端口未开放，需要配置安全组/防火墙。

### 问题 3：服务正常但无法访问

可能是云服务商的限制，联系客服确认是否有流量封堵。
