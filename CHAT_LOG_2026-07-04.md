### 阶段 2：问题修复

#### 问题 1：展开功能不工作
**原因**：
- `onclick` 属性在动态生成的 HTML 中可能失效
- `classList` 拼写错误（误写为 `classlist`）
- CSS 选择器问题

**解决方案**：
- 改用 `addEventListener` 绑定事件
- 使用 `data-idx` 传递索引
- 在 `renderLogs()` 函数末尾统一绑定事件

#### 问题 2：数据乱码
**原因**：
- API 返回二进制数据导致 JSON 序列化问题
- 代理捕获的 HTTPS 数据包含二进制内容

**解决方案**：
- 创建 `web-server-fixed.js`
- 清理不可打印字符：`replace(/[\x00-\x08\x0B\x0C\x0E-\x1F\x7F-\x9F]/g, '')`
- 过滤二进制数据，只保留可读文本

#### 问题 3：界面简陋
**原因**：
- 原版界面功能简单，视觉不够专业

**解决方案**：
- 实现 Proxyman 风格界面 (`proxyman-style.html`)
- 深色主题（VS Code 风格）
- 彩色 HTTP 方法标签
- 4 个标签页（Overview/Request/Response/Raw Data）
- 请求头、Query 参数、请求体、响应体分区域展示

#### 问题 4：浏览器缓存问题
**原因**：
- 浏览器缓存了旧版 HTML 文件
- 用户访问时看不到更新

**解决方案**：
- 清除浏览器缓存（Ctrl + Shift + Delete）
- 使用隐私模式（Ctrl + Shift + N）
- 添加随机参数（`?t=123456`）
- 创建测试页面（`test-proxyman.html`）

#### 问题 5：代理服务启动失败
**原因**：
- `server.js` 的 URL 解析有 bug
- `req.url` 可能只是路径（如 `/`），不是完整 URL

**解决方案**：
- 修复 `server.js` 第 119 行
- 添加完整 URL 构造逻辑：
  ```javascript
  let fullUrl = req.url;
  if (!fullUrl.startsWith('http')) {
    const host = req.headers.host || 'localhost';
    fullUrl = `http://${host}${req.url}`;
  }
  const url = new URL(fullUrl);
  ```

```javascript
// 移除不可打印字符
clean.responseBody = clean.responseBody.replace(
```

```javascript
// 使用 addEventListener 替代 onclick 属性
document.querySelectorAll('.capture-item').forEach(item => {
```
