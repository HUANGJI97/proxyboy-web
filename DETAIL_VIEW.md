# 📋 抓包详情展示功能说明

## 概述

Web 管理端现已支持**完整的请求和响应详情展示**，包括格式化视图和原始数据视图。

---

## ✨ 新增功能

### 1. 格式化视图（默认）

点击任意抓包记录后，展开的详情面板包含：

#### 📤 请求详情
- **请求头表格**：完整的 HTTP 请求头（Key-Value 格式）
- **原始请求数据**：针对 HTTPS 解密后的客户端数据（格式化显示）

#### 📥 响应详情
- **状态码**：彩色标签显示（2xx 绿色、3xx 黄色、4xx 红色、5xx 深红）
- **响应头表格**：完整的 HTTP 响应头
- **响应体**：完整响应内容（自动格式化，支持 JSON/HTML/文本）
- **原始响应数据**：针对 HTTPS 解密后的服务器数据

#### 📊 元信息
- **数据大小**：请求和响应的字节数
- **时间戳**：精确到毫秒

---

### 2. 原始数据视图

点击 **"📄 原始数据"** 标签页，查看：

- **完整 JSON 数据**：包含所有捕获的字段
- **格式化显示**：缩进美观，易于阅读
- **复制友好**：可直接复制粘贴用于分析

**示例：**
```json
{
  "type": "http",
  "method": "GET",
  "url": "http://example.com/",
  "requestHeaders": {
    "host": "example.com",
    "user-agent": "curl/8.5.0",
    "accept": "*/*"
  },
  "responseStatus": 200,
  "responseHeaders": {
    "date": "Sat, 04 Jul 2026 12:08:34 GMT",
    "content-type": "text/html",
    "server": "cloudflare"
  },
  "responseBody": "<!doctype html>...",
  "timestamp": "2026-07-04T12:08:34.947Z"
}
```

---

### 3. 数据大小显示

每条抓包记录展开后，顶部会显示：

```
📊 数据大小: 请求 ~342 B | 响应 ~1.23 KB
状态码: 200
```

- **请求大小**：请求头 + 请求体的总字节数
- **响应大小**：响应头 + 响应体的总字节数
- **自动格式化**：B/KB/MB 自动转换

---

### 4. 彩色状态码

响应状态码现在以**彩色标签**显示：

| 状态码 | 颜色 | 含义 |
|--------|------|------|
| 2xx | 🟢 绿色 | 成功 |
| 3xx | 🟡 黄色 | 重定向 |
| 4xx | 🔴 红色 | 客户端错误 |
| 5xx | 🔴 深红 | 服务器错误 |

---

### 5. 标签页切换

详情面板顶部有两个标签页：

1. **📋 格式化**：美观的表格和代码视图
2. **📄 原始数据**：完整的 JSON 数据

点击标签即可切换，方便不同场景查看。

---

## 🎯 使用场景

### 场景 1：调试 API 请求

1. 客户端发起 API 请求
2. 管理端自动显示抓包
3. 点击记录展开详情
4. 查看：
   - 请求头是否正确
   - 请求参数是否完整
   - 响应状态码
   - 响应体内容（JSON 格式化）

### 场景 2：分析 HTTPS 流量

1. 客户端访问 HTTPS 网站
2. 代理自动解密（需要安装 CA 证书）
3. 查看详情中的：
   - **原始请求数据**：解密后的 HTTP 请求
   - **原始响应数据**：解密后的 HTTP 响应
   - **响应体**：完整的 HTML/JSON/文本

### 场景 3：性能分析

1. 查看 **数据大小** 列
2. 识别大响应（如图片、视频、大 JSON）
3. 优化请求，减少不必要的数据传输

## 🔧 技术实现

### 前端
- **纯 JavaScript**：无依赖，轻量级
- **动态渲染**：点击展开时渲染详情
- **标签切换**：显示/隐藏不同视图
- **数据格式化**：JSON 自动缩进、HTML 转义

### 后端 API
- **/api/logs**：返回完整抓包数据
- **数据字段**：
  - `type`：http 或 https
  - `method`：GET/POST/PUT/DELETE
  - `url`：完整 URL
  - `requestHeaders`：请求头对象
  - `responseStatus`：状态码
  - `responseHeaders`：响应头对象
  - `responseBody`：响应体（字符串）
  - `clientData`：HTTPS 客户端原始数据
  - `serverData`：HTTPS 服务器原始数据
  - `timestamp`：ISO 时间戳

## ❓ 常见问题

### Q1: 响应体显示不完整？

**A**: 当前限制为 10,000 字符。如需完整数据，查看"原始数据"标签页的 JSON。

### Q2: HTTPS 数据无法解密？

**A**: 确保：
1. 客户端已安装 CA 证书
2. 证书已设为"完全信任"（iOS）
3. 代理已正确配置

### Q3: 数据大小显示不准确？

**A**: 数据大小是估算值，包含：
- 请求头 + 请求体
- 响应头 + 响应体
- 可能不包含网络连接开销
